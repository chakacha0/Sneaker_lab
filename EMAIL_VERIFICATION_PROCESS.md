# Процесс отправки email с подтверждением почты

## Описание процесса

Документ описывает полный процесс отправки email с кодом подтверждения пользователю при регистрации в системе SneakerLab.

---

## Общая схема процесса

```
Регистрация пользователя
    ↓
Генерация 6-значного кода
    ↓
Сохранение кода в БД
    ↓
Вызов send_verification_email()
    ↓
Проверка настроек SMTP
    ↓
    ├─→ SMTP не настроен → Режим разработки (вывод в консоль)
    └─→ SMTP настроен → Отправка через SMTP сервер
```

---

## Детальный процесс отправки email

### Шаг 1: Генерация кода подтверждения

**Файл:** `backend/app/repositories/users_repo.py`

```python
from app.utils.email_service import generate_verification_code

# Генерируем 6-значный код для подтверждения email
verification_code = generate_verification_code()
```

**Функция генерации:**
```python
def generate_verification_code() -> str:
    """Генерирует 6-значный код подтверждения"""
    return str(random.randint(100000, 999999))
```

**Результат:** Случайное число от 100000 до 999999 (например: `456789`)

---

### Шаг 2: Сохранение кода в базе данных

**Файл:** `backend/app/repositories/users_repo.py`

При создании пользователя код сохраняется в таблице `users`:

```sql
INSERT INTO users (role, email, password_hash, first_name, last_name, 
                   phone_number, email_verified, verification_code)
VALUES ('customer', %s, %s, %s, %s, %s, FALSE, %s)
```

**Поля:**
- `verification_code` - сохраненный 6-значный код
- `email_verified` - установлен в `FALSE` (требует подтверждения)

---

### Шаг 3: Вызов функции отправки email

**Файл:** `backend/app/routes/users.py`

После успешного создания пользователя:

```python
from app.utils.email_service import send_verification_email

# Отправляем письмо с кодом подтверждения
code = user.get("verification_code")
email_sent, email_message = send_verification_email(email, code)
```

**Параметры:**
- `email` - email адрес пользователя (например: `user@example.com`)
- `code` - 6-значный код подтверждения (например: `456789`)

**Возвращает:**
- `email_sent` (bool) - успешно ли отправлено письмо
- `email_message` (str) - сообщение о результате

---

### Шаг 4: Проверка настроек SMTP

**Файл:** `backend/app/utils/email_service.py`

Функция проверяет, настроен ли SMTP сервер:

```python
# Чтение настроек из .env файла
from app.config import settings

# Проверка наличия настроек
if not settings.SMTP_USER or not settings.SMTP_PASSWORD:
    # Режим разработки
    ...
else:
    # Реальная отправка через SMTP
    ...
```

**Настройки из `.env` файла:**
```env
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=ваш-email@gmail.com
SMTP_PASSWORD=ваш-пароль-приложения
EMAIL_FROM=ваш-email@gmail.com
```

### Шаг 5A: Режим разработки (SMTP не настроен)

**Если SMTP не настроен**, система работает в режиме разработки:

```python
if not settings.SMTP_USER or not settings.SMTP_PASSWORD:
    print(f"\n{'='*60}")
    print(f"[DEV MODE] Email не настроен - код в консоли:")
    print(f"Email: {email}")
    print(f"Код подтверждения: {code}")
    print(f"{'='*60}\n")
    return (True, f"Режим разработки. Код в консоли сервера.")
```

**Что происходит:**
1. Код выводится в консоль сервера
2. Функция возвращает `(True, "Режим разработки...")`
3. Пользователь видит сообщение: "Регистрация успешна! Проверьте консоль сервера для получения кода подтверждения."

**Пример вывода в консоль:**
```
============================================================
[DEV MODE] Email не настроен - код в консоли:
Email: user@example.com
Код подтверждения: 456789
============================================================
```

---

### Шаг 5B: Реальная отправка через SMTP (SMTP настроен)

**Если SMTP настроен**, происходит реальная отправка email:

#### 5.1. Создание сообщения

```python
from email.mime.text import MIMEText
from email.mime.multipart import MIMEMultipart

# Создаем сообщение
msg = MIMEMultipart()
msg['From'] = settings.EMAIL_FROM
msg['To'] = email
msg['Subject'] = "Код подтверждения регистрации в SneakerLab"
```

**Заголовки письма:**
- `From`: Адрес отправителя (из `EMAIL_FROM`)
- `To`: Email получателя (пользователя)
- `Subject`: Тема письма

#### 5.2. Формирование текста письма

```python
body = f"""
Здравствуйте!

Спасибо за регистрацию в SneakerLab!

Ваш код подтверждения: {code}

Введите этот код на странице подтверждения для завершения регистрации.

Код действителен в течение 10 минут.

Если вы не регистрировались на нашем сайте, просто проигнорируйте это письмо.

С уважением,
Команда SneakerLab
"""

msg.attach(MIMEText(body, 'plain', 'utf-8'))
```

**Содержание письма:**
- Приветствие
- Благодарность за регистрацию
- **Код подтверждения** (6 цифр)
- Инструкции по использованию
- Информация о сроке действия (10 минут)
- Предупреждение о безопасности

#### 5.3. Подключение к SMTP серверу

```python
import smtplib

print(f"[EMAIL] Попытка отправить письмо на {email}...")
server = smtplib.SMTP(settings.SMTP_HOST, settings.SMTP_PORT)
```

**Параметры подключения:**
- `SMTP_HOST`: Адрес SMTP сервера (например: `smtp.gmail.com`)
- `SMTP_PORT`: Порт SMTP сервера (обычно `587` для TLS)

**Для Gmail:**
- Host: `smtp.gmail.com`
- Port: `587` (TLS) или `465` (SSL)

#### 5.4. Включение шифрования (TLS)

```python
server.starttls()
print(f"[EMAIL] Подключение к {settings.SMTP_HOST}:{settings.SMTP_PORT}...")
```

**Что делает `starttls()`:**
- Включает шифрование соединения
- Защищает данные от перехвата
- Обязательно для Gmail

#### 5.5. Авторизация на SMTP сервере

```python
server.login(settings.SMTP_USER, settings.SMTP_PASSWORD)
print(f"[EMAIL] Авторизация успешна")
```

**Параметры авторизации:**
- `SMTP_USER`: Email адрес для авторизации
- `SMTP_PASSWORD`: Пароль приложения (для Gmail)

**Важно для Gmail:**
- Нельзя использовать обычный пароль аккаунта
- Нужен **"Пароль приложения"** (App Password)
- Создается в настройках Google Аккаунта

#### 5.6. Отправка письма

```python
server.send_message(msg)
server.quit()
print(f"[EMAIL] Письмо успешно отправлено на {email}")
return (True, "Письмо отправлено успешно")
```

**Что происходит:**
1. `send_message(msg)` - отправляет письмо через SMTP сервер
2. `server.quit()` - закрывает соединение
3. Возвращает успешный результат

---

### Шаг 6: Обработка ошибок

Функция обрабатывает различные типы ошибок:

#### 6.1. Ошибка авторизации SMTP

```python
except smtplib.SMTPAuthenticationError as e:
    error_msg = f"Ошибка авторизации SMTP: {str(e)}. Проверьте SMTP_USER и SMTP_PASSWORD в .env"
    print(f"[EMAIL ERROR] {error_msg}")
    return (False, error_msg)
```

**Причины:**
- Неверный `SMTP_USER` или `SMTP_PASSWORD`
- Для Gmail: используется обычный пароль вместо пароля приложения
- Двухфакторная аутентификация не включена

**Решение:**
- Проверить настройки в `.env`
- Для Gmail создать пароль приложения

#### 6.2. Общая ошибка SMTP

```python
except smtplib.SMTPException as e:
    error_msg = f"Ошибка SMTP: {str(e)}"
    print(f"[EMAIL ERROR] {error_msg}")
    return (False, error_msg)
```

**Причины:**
- Неверный `SMTP_HOST` или `SMTP_PORT`
- Проблемы с сетью
- SMTP сервер недоступен

#### 6.3. Другие ошибки

```python
except Exception as e:
    error_msg = f"Ошибка при отправке email: {str(e)}"
    print(f"[EMAIL ERROR] {error_msg}")
    return (False, error_msg)
```

**Причины:**
- Неожиданные ошибки
- Проблемы с кодировкой
- Другие системные ошибки

## Настройка SMTP для Gmail

### Шаг 1: Создание пароля приложения

1. Откройте [Google Аккаунт](https://myaccount.google.com/)
2. Перейдите в [Безопасность](https://myaccount.google.com/security)
3. Включите **Двухэтапную аутентификацию** (если не включена)
4. Перейдите в [Пароли приложений](https://myaccount.google.com/apppasswords)
5. Выберите "Почта" и "Другое устройство"
6. Введите название: "SneakerLab"
7. Нажмите "Создать"
8. Скопируйте 16-значный пароль (например: `abcd efgh ijkl mnop`)

### Шаг 2: Настройка .env файла

Создайте или откройте файл `backend/.env`:

```env
# SMTP настройки для отправки email
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=ваш-email@gmail.com
SMTP_PASSWORD=abcdefghijklmnop  # Пароль приложения (без пробелов)
EMAIL_FROM=ваш-email@gmail.com
```

**Важно:**
- `SMTP_PASSWORD` - это пароль приложения, а не обычный пароль
- Уберите пробелы из пароля приложения (если есть)
- Перезапустите сервер после изменения `.env`

## Безопасность

1. **Шифрование соединения:** Используется TLS (`starttls()`)
2. **Пароль приложения:** Для Gmail используется специальный пароль, а не основной
3. **Код в БД:** Код хранится в базе данных и сравнивается при подтверждении
4. **Срок действия:** Код действителен 10 минут (логика в приложении)
5. **Одноразовость:** После подтверждения код удаляется из БД

## Резюме

Процесс отправки email с подтверждением включает:

1. **Генерацию** 6-значного кода
2. **Сохранение** кода в базе данных
3. **Проверку** настроек SMTP
4. **Два режима работы:**
   - Режим разработки (вывод в консоль)
   - Реальная отправка через SMTP
5. **Формирование** письма с кодом
6. **Отправку** через SMTP сервер (Gmail, Mailgun и др.)
7. **Обработку** ошибок и логирование

Система автоматически переключается между режимами в зависимости от наличия настроек SMTP, что удобно для разработки и продакшена.
