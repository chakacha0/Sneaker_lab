# Блок-схема алгоритма создания заказа

## Алгоритм: Создание заказа (Order Creation)

### Описание
Алгоритм описывает процесс оформления заказа пользователем: от заполнения формы адреса до создания заказа в базе данных, обновления склада и очистки корзины.

## Текстовое описание алгоритма

### Шаг 1: Начало
Пользователь заполняет форму адреса и нажимает кнопку "Place Order"

### Шаг 2: Валидация формы
- Проверка обязательных полей (country, city, street, house)
- Если валидация не пройдена → вывод ошибки → КОНЕЦ

### Шаг 3: Создание адреса
- Вызов `createAddress(addressData)`
- Если ошибка → вывод ошибки → КОНЕЦ
- Получение `address_id`

### Шаг 4: Получение корзины
- Вызов `get_or_create_cart(user_id)`
- Получение `cart_id`

### Шаг 5: Получение товаров из корзины
- Вызов `get_cart_items(cart_id)`
- Если корзина пуста → вывод ошибки "Корзина пуста" → КОНЕЦ

### Шаг 6: Вычисление исходной стоимости
```
original_total = Σ(item.price × item.quantity)
для всех item в cart_items
```

### Шаг 7: Обработка промокода
- Если промокод указан:
  - Получение промокода из БД: `get_promo_code_by_code(code)`
  - Если найден → получение `promo_id`
  - Вычисление итоговой стоимости: `calculate_order_total_with_promo(original_total, code)`
- Если промокод не указан:
  - `promo_id = NULL`
  - `final_total = original_total`

### Шаг 8: Создание заказа
- INSERT INTO orders (user_id, address_id, promo_id, total_price)
- Получение `order_id`

### Шаг 9: Цикл обработки товаров
Для каждого товара в корзине:
1. Создание записи в order_items
2. Уменьшение количества на складе (UPDATE product_stock)

### Шаг 10: Обновление счетчика промокода
- Если `promo_id` не NULL:
  - UPDATE promo_codes SET used_count = used_count + 1

### Шаг 11: Очистка корзины
- DELETE FROM cart_items WHERE cart_id = cart_id

### Шаг 12: Фиксация транзакции
- COMMIT транзакции

### Шаг 13: Получение полной информации о заказе
- Вызов `get_order_by_id(order_id)`

### Шаг 14: Вывод сообщения об успехе
- "Order successfully placed!"

### Шаг 15: Закрытие модального окна
- Закрытие через 1.5 секунды

### Обработка ошибок
- При любой ошибке → ROLLBACK транзакции
- Вывод сообщения об ошибке
- КОНЕЦ

---

## Сложность алгоритма

- **Временная сложность:** O(n), где n - количество товаров в корзине
- **Пространственная сложность:** O(1) - константная память

## Особенности алгоритма

1. **Транзакционность:** Все операции выполняются в одной транзакции
2. **Атомарность:** Либо все операции выполняются успешно, либо откатываются все изменения
3. **Проверка наличия:** Перед созданием заказа проверяется наличие товаров в корзине
4. **Обновление склада:** Автоматическое уменьшение количества товаров на складе
5. **Очистка корзины:** Автоматическая очистка корзины после успешного создания заказа
