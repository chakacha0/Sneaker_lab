# Пошаговая инструкция: Настройка отправки Email

## Быстрый старт

### Вариант 1: Режим разработки (без настройки email)

Если вы просто хотите протестировать функционал, **ничего делать не нужно!**

Система автоматически будет выводить ссылку подтверждения в консоль сервера при регистрации пользователя.

Вы увидите что-то вроде:
```
[DEV MODE] Verification link for user@example.com: http://localhost:5173/verify-email?token=...
```

### Вариант 2: Настройка реальной отправки email (Gmail)

#### Шаг 1: Создайте файл `.env`

1. Откройте папку `backend/`
2. Создайте новый файл с именем `.env` (с точкой в начале!)
3. Если файл уже существует, откройте его

#### Шаг 2: Добавьте настройки в `.env`

Скопируйте и вставьте в файл `.env`:

```env
# Database settings
DB_HOST=localhost
DB_PORT=5432
DB_NAME=shoeses_shop
DB_USER=postgres
DB_PASSWORD=postgre

# SMTP настройки для отправки email
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=ваш-email@gmail.com
SMTP_PASSWORD=ваш-пароль-приложения
EMAIL_FROM=ваш-email@gmail.com
FRONTEND_URL=http://localhost:5173
```

#### Шаг 3: Получите пароль приложения Gmail

**Важно:** Gmail не позволяет использовать обычный пароль. Нужен "Пароль приложения".

1. Откройте браузер и войдите в [Google Аккаунт](https://myaccount.google.com/)

2. Перейдите в [Безопасность](https://myaccount.google.com/security)

3. **Включите двухфакторную аутентификацию** (если еще не включена):
   - Нажмите "Двухэтапная аутентификация"
   - Следуйте инструкциям

4. Создайте пароль приложения:
   - В разделе "Безопасность" найдите **"Пароли приложений"**
   - Если не видите, попробуйте прямой переход: [Пароли приложений](https://myaccount.google.com/apppasswords)
   - Выберите приложение: **"Почта"**
   - Выберите устройство: **"Другое (указать название)"**
   - Введите: `SneakerLab`
   - Нажмите **"Создать"**

5. **Скопируйте 16-символьный пароль** (он будет выглядеть как: `abcd efgh ijkl mnop`)
   - Удалите все пробелы: `abcdefghijklmnop`
   - Это и есть ваш `SMTP_PASSWORD`

#### Шаг 4: Обновите `.env` файл

Замените в файле `backend/.env`:

```env
SMTP_USER=ваш-реальный-email@gmail.com
SMTP_PASSWORD=abcdefghijklmnop  # пароль приложения БЕЗ пробелов
EMAIL_FROM=ваш-реальный-email@gmail.com
```

**Пример реального файла:**
```env
SMTP_USER=john.doe@gmail.com
SMTP_PASSWORD=abcd efgh ijkl mnop
EMAIL_FROM=john.doe@gmail.com
```

#### Шаг 5: Перезапустите сервер

После изменения `.env` файла, перезапустите backend сервер, чтобы изменения вступили в силу.

## Проверка работы

1. Зарегистрируйте нового пользователя
2. Проверьте почту (включая папку "Спам")
3. Перейдите по ссылке в письме
4. Email должен подтвердиться

## Альтернативные email провайдеры

### Outlook/Hotmail
```env
SMTP_HOST=smtp-mail.outlook.com
SMTP_PORT=587
```

### Yahoo
```env
SMTP_HOST=smtp.mail.yahoo.com
SMTP_PORT=587
```

### Другие провайдеры
Используйте настройки SMTP вашего провайдера. Обычно их можно найти в настройках почтового ящика.

## Устранение проблем

### Ошибка "Authentication failed"
- Убедитесь, что используете пароль приложения, а не обычный пароль
- Проверьте, что двухфакторная аутентификация включена

### Письма не приходят
- Проверьте папку "Спам"
- Убедитесь, что `FRONTEND_URL` правильный
- Проверьте логи сервера на наличие ошибок

### Пароли приложений не доступны
- Убедитесь, что двухфакторная аутентификация включена
- Некоторые корпоративные аккаунты Google могут не поддерживать пароли приложений
