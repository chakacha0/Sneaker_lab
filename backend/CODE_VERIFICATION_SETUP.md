# Настройка подтверждения Email через код

## Что изменилось

Система теперь работает с **6-значным кодом подтверждения** вместо ссылки:
- Пользователь регистрируется
- Получает код на email (например: `123456`)
- Вводит код на странице подтверждения
- Email подтверждается

## Шаг 1: Применить миграцию базы данных

Выполните SQL скрипт для обновления структуры базы данных:

```sql
-- Переименовываем колонку (если она существует)
ALTER TABLE users 
RENAME COLUMN email_verification_token TO verification_code;

-- Если колонка не существовала, создаем новую
ALTER TABLE users 
ADD COLUMN IF NOT EXISTS verification_code VARCHAR(10);

-- Обновляем индекс
DROP INDEX IF EXISTS idx_email_verification_token;
CREATE INDEX IF NOT EXISTS idx_verification_code ON users(verification_code);
```

Или выполните файл миграции:
```bash
psql -U postgres -d shoeses_shop -f migration_code_verification.sql
```

## Шаг 2: Перезапустите сервер

После применения миграции перезапустите backend сервер.

## Как это работает

1. **Регистрация:**
   - Пользователь заполняет форму регистрации
   - Генерируется 6-значный код (например: `456789`)
   - Код сохраняется в базе данных
   - Код отправляется на email пользователя

2. **Подтверждение:**
   - Пользователь автоматически переходит на страницу ввода кода
   - Вводит 6-значный код из письма
   - Система проверяет код
   - Если код верный → email подтверждается

3. **Режим разработки:**
   - Если SMTP не настроен, код выводится в консоль сервера
   - Удобно для тестирования без настройки email

## Пример письма

```
Здравствуйте!

Спасибо за регистрацию в SneakerLab!

Ваш код подтверждения: 456789

Введите этот код на странице подтверждения для завершения регистрации.

Код действителен в течение 10 минут.

Если вы не регистрировались на нашем сайте, просто проигнорируйте это письмо.

С уважением,
Команда SneakerLab
```

## Настройка SMTP (опционально)

Для отправки реальных писем настройте `.env` файл (см. `SETUP_EMAIL_GUIDE.md`).

Если SMTP не настроен, код будет выводиться в консоль сервера при регистрации.
