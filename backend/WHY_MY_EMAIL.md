# Зачем нужен ваш Gmail для отправки писем?

## Простыми словами:

**Ваш Gmail аккаунт = это "почтовый ящик", через который отправляются письма всем пользователям**

## Как это работает:

```
Ваш Gmail (SMTP_USER) 
    ↓
Используется как "отправитель" для ВСЕХ писем
    ↓
Письма отправляются ОТ имени EMAIL_FROM
    ↓
Пользователи видят EMAIL_FROM в поле "От кого"
```

## Важные моменты:

### 1. **SMTP_USER** - это технический аккаунт для отправки
- Это ваш Gmail, который используется для **авторизации** в Gmail сервере
- Это как "логин" для доступа к отправке писем
- **Пользователи НЕ видят этот email!**

### 2. **EMAIL_FROM** - это то, что видят пользователи
- Это адрес, который пользователи увидят в поле "От кого"
- Можете указать любой адрес (даже несуществующий, но лучше реальный)
- Например: `noreply@sneakerlab.com` или `support@sneakerlab.com`

### 3. Что видят пользователи:

Когда пользователь получает письмо, он видит:
```
От: support@sneakerlab.com  (это EMAIL_FROM)
Кому: user@example.com      (email пользователя)
Тема: Подтверждение регистрации в SneakerLab
```

**Они НЕ видят ваш личный Gmail!**

## Альтернативы:

### Вариант 1: Использовать отдельный Gmail для проекта
Создайте отдельный Gmail аккаунт специально для проекта:
- `sneakerlab.project@gmail.com` - для SMTP_USER
- `noreply@sneakerlab.com` - для EMAIL_FROM

### Вариант 2: Режим разработки (БЕЗ настройки)
Просто оставьте SMTP_USER и SMTP_PASSWORD пустыми:
```env
SMTP_USER=
SMTP_PASSWORD=
EMAIL_FROM=
```

Тогда ссылки будут выводиться в консоль сервера - удобно для тестирования!

### Вариант 3: Использовать профессиональный email сервис
- SendGrid
- Mailgun
- Amazon SES
- Mailjet

Эти сервисы специально созданы для отправки транзакционных писем.

## Почему Gmail требует пароль приложения?

Gmail защищает ваш аккаунт:
- Обычный пароль не работает для программ
- Пароль приложения - это специальный ключ только для отправки писем
- Если кто-то узнает пароль приложения, он не сможет войти в ваш Gmail

## Итого:

✅ **SMTP_USER** - ваш Gmail (технический, пользователи не видят)
✅ **EMAIL_FROM** - адрес, который видят пользователи (может быть другим)
✅ Можно использовать отдельный Gmail для проекта
✅ Можно вообще не настраивать и использовать режим разработки
