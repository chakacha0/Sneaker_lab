# Решение проблем с отправкой Email

## Письма не приходят - что делать?

### Шаг 1: Проверьте консоль сервера

**Самое важное!** Откройте консоль, где запущен backend сервер, и посмотрите на сообщения.

#### Если видите:
```
[DEV MODE] Email не настроен - ссылка в консоли:
Email: user@example.com
Ссылка: http://localhost:5173/verify-email?token=...
```

**Решение:** Это нормально! SMTP не настроен, система работает в режиме разработки.
- Скопируйте ссылку из консоли
- Вставьте в браузер
- Email подтвердится

#### Если видите:
```
[EMAIL ERROR] Ошибка авторизации SMTP: ...
```

**Решение:** 
1. Проверьте файл `.env` в папке `backend/`
2. Убедитесь, что `SMTP_USER` и `SMTP_PASSWORD` заполнены
3. Для Gmail используйте **пароль приложения**, а не обычный пароль
4. Перезапустите сервер после изменения `.env`

#### Если видите:
```
[EMAIL] Письмо успешно отправлено на user@example.com
```

**Но письмо не пришло?** См. Шаг 2.

### Шаг 2: Проверьте папку "Спам"

**90% проблем решается здесь!**

1. Откройте почтовый ящик
2. Проверьте папку **"Спам"** или **"Нежелательная почта"**
3. Если письмо там - отметьте как "Не спам"

### Шаг 3: Проверьте настройки `.env`

Откройте файл `backend/.env` и проверьте:

```env
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=ваш-email@gmail.com        # ← Должен быть заполнен
SMTP_PASSWORD=ваш-пароль-приложения  # ← Должен быть заполнен (16 символов без пробелов)
EMAIL_FROM=ваш-email@gmail.com       # ← Должен быть заполнен
FRONTEND_URL=http://localhost:5173
```

**Важно:**
- Если `SMTP_USER` или `SMTP_PASSWORD` пустые → система работает в режиме разработки
- Для Gmail нужен **пароль приложения**, не обычный пароль
- После изменения `.env` **перезапустите сервер**

### Шаг 4: Проверьте пароль приложения Gmail

Если используете Gmail:

1. Убедитесь, что двухфакторная аутентификация включена
2. Создайте новый пароль приложения: [Пароли приложений](https://myaccount.google.com/apppasswords)
3. Скопируйте 16 символов **БЕЗ пробелов**
4. Вставьте в `SMTP_PASSWORD` в `.env`
5. Перезапустите сервер

### Шаг 5: Проверьте логи сервера

При регистрации пользователя в консоли должно быть:

**Успешная отправка:**
```
[EMAIL] Попытка отправить письмо на user@example.com...
[EMAIL] Подключение к smtp.gmail.com:587...
[EMAIL] Авторизация успешна
[EMAIL] Письмо успешно отправлено на user@example.com
```

**Ошибка:**
```
[EMAIL ERROR] Ошибка авторизации SMTP: ...
```

### Шаг 6: Проверьте email адрес пользователя

Убедитесь, что:
- Email адрес правильный (без опечаток)
- Email адрес существует
- Вы проверяете правильный почтовый ящик

### Шаг 7: Попробуйте другой email провайдер

Если Gmail не работает, попробуйте:

**Outlook/Hotmail:**
```env
SMTP_HOST=smtp-mail.outlook.com
SMTP_PORT=587
```

**Yahoo:**
```env
SMTP_HOST=smtp.mail.yahoo.com
SMTP_PORT=587
```

## Быстрая диагностика

### Вопрос 1: Видите ли вы в консоли `[DEV MODE]`?
- **Да** → SMTP не настроен, используйте ссылку из консоли
- **Нет** → Переходите к вопросу 2

### Вопрос 2: Видите ли вы `[EMAIL ERROR]`?
- **Да** → Проверьте настройки `.env` и пароль приложения
- **Нет** → Переходите к вопросу 3

### Вопрос 3: Видите ли вы `[EMAIL] Письмо успешно отправлено`?
- **Да** → Проверьте папку "Спам"
- **Нет** → Проверьте, что сервер запущен и `.env` файл существует

## Режим разработки (без настройки email)

Если не хотите настраивать email сейчас:

1. Оставьте в `.env` пустые значения:
```env
SMTP_USER=
SMTP_PASSWORD=
EMAIL_FROM=
```

2. При регистрации ссылка будет выводиться в консоль сервера
3. Скопируйте ссылку и вставьте в браузер

Это удобно для разработки и тестирования!

## Все еще не работает?

1. Проверьте, что backend сервер запущен
2. Проверьте, что файл `.env` находится в папке `backend/`
3. Проверьте, что после изменения `.env` вы перезапустили сервер
4. Проверьте консоль сервера на наличие ошибок
5. Попробуйте зарегистрировать пользователя снова и посмотрите логи
