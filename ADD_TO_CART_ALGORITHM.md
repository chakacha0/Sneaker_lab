# Блок-схема алгоритма добавления товара в корзину

## Алгоритм: Добавление товара в корзину (Add to Cart)

### Описание
Алгоритм описывает процесс добавления товара в корзину пользователя: от нажатия кнопки на странице товара до сохранения товара в базе данных с проверкой наличия на складе и обновлением количества, если товар уже есть в корзине.

## Текстовое описание алгоритма

### Шаг 1: Начало
Пользователь на странице товара нажимает кнопку "Add to Cart"

### Шаг 2: Проверка выбранного размера
- Проверка: `selectedSize === null?`
- Если размер не выбран → вывод сообщения "Please select a size" → КОНЕЦ

### Шаг 3: Проверка авторизации
- Получение пользователя из `localStorage`
- Если пользователь не авторизован → вывод сообщения "Please log in to add items to cart" → КОНЕЦ

### Шаг 4: Инициализация процесса
- Установка `setIsAdding(true)` - флаг загрузки
- Очистка сообщений: `setMessage("")`

### Шаг 5: Отправка HTTP запроса
- POST запрос на `/cart/add`
- Данные: `{user_id, product_id, size, quantity}`

### Шаг 6: Валидация на сервере
- Проверка наличия обязательных полей: `user_id`, `product_id`, `size`
- Если данные невалидны → HTTPException 400 → КОНЕЦ

### Шаг 7: Получение или создание корзины
- Вызов `get_or_create_cart(user_id)`
- Если корзина не существует → создание новой корзины
- Получение `cart_id`

### Шаг 8: Проверка наличия товара на складе
- Вызов `get_product_stock_by_size(product_id, size)`
- Получение `available_quantity` - доступного количества на складе

### Шаг 9: Проверка наличия товара в корзине
- Поиск товара в корзине с таким же `product_id` и `size`
- Если товар найден → получение текущего количества `current_cart_quantity`
- Если товар не найден → `current_cart_quantity = 0`

### Шаг 10: Вычисление нового количества
```
new_quantity = current_cart_quantity + quantity
```

### Шаг 11: Проверка доступного количества
- Сравнение: `new_quantity > available_quantity?`
- Если превышает → ValueError "Недостаточно товара на складе. Доступно: X, запрашивается: Y" → HTTPException 400 → КОНЕЦ

### Шаг 12: Добавление или обновление товара
- **Если товар уже в корзине:**
  - UPDATE `cart_items` SET `quantity = new_quantity` WHERE `cart_item_id`
- **Если товар не в корзине:**
  - INSERT INTO `cart_items` (`cart_id`, `product_id`, `size`, `quantity`)

### Шаг 13: Фиксация транзакции
- COMMIT транзакции в базе данных

### Шаг 14: Возврат результата
- Возврат объекта: `{message: "Товар добавлен в корзину", item: {...}}`

### Шаг 15: Обработка результата на клиенте
- **Если успешно:**
  - Вывод сообщения "Item added to cart!"
  - Автоматическое скрытие сообщения через 2 секунды
- **Если ошибка:**
  - Вывод сообщения об ошибке: `error.message`

### Шаг 16: Завершение
- Сброс флага загрузки: `setIsAdding(false)`
- КОНЕЦ

## Сложность алгоритма

- **Временная сложность:** O(1) - константная, так как все операции с базой данных выполняются по индексам
- **Пространственная сложность:** O(1) - константная память

## Особенности алгоритма

1. **Проверка авторизации:** Товар можно добавить только авторизованному пользователю
2. **Проверка размера:** Обязательный выбор размера перед добавлением
3. **Проверка наличия на складе:** Автоматическая проверка доступного количества
4. **Объединение товаров:** Если товар с таким же размером уже в корзине, увеличивается количество вместо создания новой записи
5. **Валидация количества:** Проверка, что суммарное количество (текущее + новое) не превышает доступное на складе
6. **Атомарность:** Все операции выполняются в одной транзакции
7. **Обратная связь:** Пользователь получает сообщение об успехе или ошибке

## Примеры сценариев

### Сценарий 1: Добавление нового товара
- Товар не в корзине
- На складе есть 10 единиц
- Запрашивается 1 единица
- **Результат:** Товар добавлен в корзину (quantity = 1)

### Сценарий 2: Добавление товара, который уже в корзине
- Товар уже в корзине (quantity = 2)
- На складе есть 10 единиц
- Запрашивается еще 3 единицы
- **Результат:** Количество обновлено (quantity = 5)

### Сценарий 3: Превышение доступного количества
- Товар уже в корзине (quantity = 8)
- На складе есть 10 единиц
- Запрашивается еще 5 единиц
- **Результат:** Ошибка "Недостаточно товара на складе. Доступно: 10, запрашивается: 13"

### Сценарий 4: Товар отсутствует на складе
- На складе 0 единиц
- Запрашивается 1 единица
- **Результат:** Ошибка "Недостаточно товара на складе. Доступно: 0, запрашивается: 1"
